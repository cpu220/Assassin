node全局命令

  • 有没有这种感觉：写的脚手架，工具太多了，每次都要拖工程，或者把什么文件往什么工具目录下放，然后再敲命令行执行，是在是太麻烦了。
  • 所以，为什么不自己写一个command，专门为自己服务的控制台，想要什么功能自己可以随时添加命令挂载？
  • 本文以我自己的command为demo，懒得看下面图文的可以直接看我的妲己

解说

  • 我其实用到了commander这个包，详细可以npm上搜下他对应的api。

  • 为了能够让我们自己的command变为全局，所以需要在 package里添加一项

  • 所有文件第一行都要加一行下面的命令，意思是为了确定node安装在 /usr/bin路径里，系统看到后会到env里找node的安装路径，为了找到对应路径下的解析器来完成操作。

  • const program = require('commander') ,这个包就是贯穿我们整个脚本的控制台了。下面简单介绍下主要用到的api，具体使用可以看官网，绝对比我写的详细。

    • version() // 版本号，可以把package给require进来，这样就只用在一个地方同步版本号了
      • help（）// 输出帮助信息
      • command（） // 定义一个命令名字
      • alias() // 命令对应的缩写
      • description() // 命令的描述说明
      • action(function(cmd,opations){}) // 注册一个命令的回调事件
      • parse(process.argv) // 解析参数

发布

  • 我们已经按目录格式、以及命令对应的api都写好了的时候。这时候需要发布。
    • git push 到自己的仓库里
    • npm publish 到npm 上
    • 由于国内的cnpm是10分钟同步一次，所以想尽快体验下自己写的command，需要用npm来安装。
    • linux 环境需要先 npm install -g xxx 之后，还需要以下命令配置到全局才能使用

总结

其实就是利用了commander已经封装好的接口，来组装脚本。在通过package.bin 来映射主文件。依靠#!/usr/bin/env node 来进行识别解析。这样达到全局执行脚本接口。写的不是很详细，后面待补充

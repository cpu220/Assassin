Linux下node环境配置及安装

  • 我们都知道进行node开发，是需要node环境。那么一套标准的前端环境最低也是要包含node，git，npm 这3个。那么我们第一步就要先进行环境及工具的安装。

node安装

  • 我自己用于调试代码的文件上传工具,其实随便下个sftp工具就可以了。
  • linux下node安装和mac／windows 不一样。说是安装，其实是配置全局参数。
  • 先node官网下载安装包,见下图

  • 然后将安装包扔到服务器指定的’xxx’目录下（目录随便定），然后解压缩

    tar xf [node安装包名]

  • 完成后会看到多了一个目录,为了方便，我将其命名为nodejs，不过事例起名为[node123],见下图

  • node文件已经在服务器上了，然而我们需要配置全局，才可以在任何地方执行node命令。所以需要下面的命令

  • 截止目前，我们已经成功配置来node／npm 环境。可以输入node/npm 来验证一下。为了安装的快点，可以安装下cnpm。

  • 需要注意的是，任何npm install -g 安装的工具，都会安装在/xxx/nodejs/bin/下。然后通过全局配置来达到安装的效果。

Hexo 安装

  • 接上面流程，我们已经成功安装来node和npm。那么怎么安装node其他工具，下面以hexo为例。

  • 安装完毕。其实看到下图也能理解来

  • 那么我们来实验下我们安装的hexo怎么样
    • 已经能跑起来了，虽然提示的地址是0.0.0.0 。但是这地址是相对于当前服务器的本地地址，但对外访问，就是当前的服务器ip地址。
  • 然后，让我们去访问的时候，浏览器提示禁止访问。原因=> 需要关闭防火墙 service firewalld stop 。 不同服务器命令不同
  • 这下终于可以看到Hexo在服务器上跑起来来，就跟在本地一样。但我们关闭来终端后，却发现又无法访问来。什么原因呢？
    • 原因就在于，我们访问服务器，其实可以理解为开来一个窗口去访问。但当我们断开窗口来，服务也就随之终止。所以我们需要将服务跑在后台进程中，这样就不会随终端关闭而关闭。
      • 先将命令hexo clean && hexo s 放在一个 build.sh 的文件内，再将该文件上传到hexo根目录下。
      • 新上传的文件其实是没有权限去执行的，所以需要开执行权限，这跟是不是root权限没关系。
      • chmod 777 build.sh
      • 执行后台命令 : nohup ./build.sh &
      • 这下可以关闭终端来，而且服务也没有停,于是我可以把地址发给朋友们吹牛[哔–]了。

git安装

  • 我们已经配置好来环境，也将Hexo跑来起来。但我们每次更新都要将文件update到服务器上，太麻烦了。所以我们希望可以让服务器上的markdown文件能够随某种机制更新。我们先按最简单的实现下，即手动git pull 来更新。但linux是没有git工具的，所以我们需要先安装。
  • 这里我尝试来几种安装方式，yum安装是最省事的

  • 首先将yum升级下，据说是因为服务器镜像版本较低

  • 安装过程中可能会在某几个阶段卡死一段时间，别ctrl+c去中断。等等就好。更新成功后,再安装依赖包

  • 安装git

  • 好了，我们现在终于可以再Linux 服务器上去 git pull来更新

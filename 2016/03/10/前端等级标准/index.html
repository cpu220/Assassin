前端等级评定标准


入门

  • html方面：知道标准document的结构，能说对 script link 标签的正常参数,知道cdn，但可以未接触过
  • css方面：position／display／overflower是做什么
  • javascript：只要选择器，事件绑定
  • html:知道document／xml的文本规则,知道行内块状元素
  • 能够脱离dreamweaver，而使用sublime／webStrom／visual code／eclipse/ATOM等编辑平台进行正规开发

初级

  • 知道前端是做什么，知道前端能够做什么（范围），对于常见页面特效有一定思路去实现，常见浏览器兼容问题进行处理（少部分）
  • JS
    • 基本掌握Jquery／bootstrap／backbone等前端主流框架的一种，（会查API，但常用的不需要），对于日常开发开发能够胜任，可以基本没有自己的原创代码，能够对于前端控件采取网上搜第三方控件为主，自己有一定能力去修改
  • html
    • 未接触过移动端h5开发，但知道h5能做什么。可以没做过，但知道html5、html4.0.1、xhtml1.0的差异，知道盒模型，在不考虑性能的前提下能够按视觉稿准确重构出对应页面（不允许有1px的误差）
  • css
    • 对于常用css有哪些，且对应的参数能说出来。可以没做过，但知道css3.0与css2.0的差异

中级

  • 有了解过Jquery／bootstrap／underscore等前端框架/库至少一种源码，能知道并使用prototype（js原型链），能说出面向对象思想并体现在自己的代码里， 知道js模块化开发，会用但不熟练 。
  • 知道SEO，能说出前端优化的有效方案至少3条
  • 知道nondejs，可以未用过但必须了解过
  • 使用过glup、grunt等，
  • 接触过移动端开发，会一点h5+css3的动画，允许借助API手册
  • 能说出html冒泡的处理方案至少2种
  • 常用前端与服务端数据交互至少知道以下一种
    • 同步，submit方式，及数据格式，可以不清楚原理
    • 异步，ajax方式，及数据格式，可以不清楚原理
    • 同步和异步的根本原因及实现，能说出大概即可
  • 能够独立完成自己手上的需求而无任何问题硬性要求

高级

  • 代码托管svn／github／coding／jsFiddle／RunJS 知道并且使用过其中2种（svn是必须知道）
  • 能够同时负责1个或多个项目并承担owner的角色，在开发过程中对项目的前端框架及组件进行基础建设。硬性要求
    • 前端框架使用及搭建，前端基础建设
    • css样式库的搭建及完善
  • 代码开发模块化，seajs／RequireJS 都会，且至少掌握其中一种，有自己的源码，能够熟练的进行前端组件开发
    • 对于控件（css＋html＋js）的实现建模有清晰明确的思路并规范
  • nodejs开发
  • 能够对前端数据展示／交互／结构的处理及优化
  • 知道webview及其性能和原理

资深

  • 能够承担团队TL的角色硬性要求
  • 至少会一门非前端语法，对待问题的解决思路不局限于前端技术手段的范围内
  • 能够熟练的开发工具以提高日常开发工作效率，优化工作流程
  • 对系统整站的数据链路有清晰的了解，对客户端开发有一定的了解，

加分项：

  • 熟练使用git，nonde
    • 有开发过nodejs工具
  • 对于所掌握的技术能有自己的见解，能够问为什么？知其然且知其所以然
  • 有自己的源码及博客或者站
  • 有ios客户端开发经验

能力标准的理解

  • 了解 ：听说过，能说出该技术是做什么，为了解决什么
  • 熟练 ：不需要使用API，API的作用是变成了对该技能理解的提纲
  • 掌握 ：能够知道该技术的机制及原理
  • 精通 ：能够说出该技术在计算机编译的过程
